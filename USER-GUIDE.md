## Что такое SmartCon

SmartCon — плагин для Autodesk Revit, который автоматизирует рутинные операции по соединению MEP-элементов (трубы, арматура, фитинги).

**Главная функция — PipeConnect:**
Соединение двух элементов в 3D-виде двумя кликами мыши с автоматическим:
- выравниванием позиции и поворота
- подбором типоразмера по диаметру
- вставкой переходников при несовпадении размеров
- подбором фитинга при несовпадении типов соединений
- перемещением целой цепочки элементов как единого целого

## Установка

### Вариант 1: Инсталлятор (рекомендуется)

1. Скачайте `SmartCon-Setup.exe` со страницы [Releases](https://github.com/Alexandrisius/AGK-SmartCon-Pro/releases)
2. Запустите установщик (права администратора не требуются)
3. Установщик автоматически определит, какие версии Revit установлены на компьютере
4. Нажмите «Установить»
5. Перезапустите Revit

### Вариант 2: Из исходников (для разработчиков)

```bash
git clone https://github.com/Alexandrisius/AGK-SmartCon-Pro.git
cd AGK-SmartCon-Pro
build-and-deploy.bat
```

**Требования:** .NET 8.0 SDK, Visual Studio 2022 или Rider.

### Проверка установки

После перезапуска Revit на ленте должна появиться вкладка **SmartCon** с кнопками:
- **PipeConnect** — соединение элементов
- **Settings** — настройки типов и маппинга
- **About** — информация о версии

## Первый запуск — настройка

> ⚠️ **Перед первым использованием PipeConnect** необходимо настроить типы коннекторов и правила маппинга. Без этого плагин не сможет подбирать фитинги.

### Шаг 1: Откройте настройки

Нажмите кнопку **Settings** на ленте SmartCon. Откроется немодальное окно «Настройки SmartCon» с двумя вкладками.

### 4.1 Типы коннекторов

Вкладка **«Типы коннекторов»** — здесь вы задаёте, какие типы соединений используются в вашем проекте.

```
┌──────────────────────────────────────────────────┐
│ Код | Название      | Описание                    │
│-----|---------------|-----------------------------│
│  1  | Сварка        | Сварное соединение          │
│  2  | Резьба        | Резьбовое соединение        │
│  3  | Раструб       | Раструбное соединение       │
│  4  | Фланец        | Фланцевое соединение        │
│  5  | Пресс-фитинг  | Обжимное соединение         │
└──────────────────────────────────────────────────┘
 [+ Добавить]  [Удалить]  [Сохранить]
```

**Как заполнить:**

1. Нажмите **«+ Добавить»**
2. Введите **Код** — числовой идентификатор (1, 2, 3...)
3. Введите **Название** — например «Сварка»
4. Введите **Описание** — опционально, для пояснения
5. Повторите для каждого типа соединения в проекте
6. Нажмите **«Сохранить»**

> **Совет:** Код типа записывается в `Description` коннектора семейства. Используйте простые числа (1, 2, 3...) — они будут храниться прямо в Revit-файле.

### 4.2 Правила маппинга фитингов

Вкладка **«Правила маппинга»** — здесь вы задаёте, какие фитинги использовать для соединения разных типов.

```
┌──────────────────────────────────────────────────────┐
│ От типа: [Сварка ▼]    К типу: [Резьба ▼]           │
│ [ ] Совместимы без фитинга                           │
│                                                      │
│ Семейства фитингов:                                  │
│ Приоритет | Семейство           | Типоразмер         │
│ --------- | ------------------- | ------------------ │
│     1     | Переходник_С-Р      | *                  │
│     2     | Муфта_универсальная | *                  │
│                                                      │
│ [+ Добавить семейство]  [Выбрать...]                 │
│                                                      │
│ [Сохранить]                                          │
└──────────────────────────────────────────────────────┘
```

**Как заполнить:**

1. Выберите **«От типа»** — тип соединения первого коннектора
2. Выберите **«К типу»** — тип соединения второго коннектора
3. **Если типы одинаковые** (сварка → сварка):
   - Поставьте галочку **«Совместимы без фитинга»**
   - Добавьте семейства **переходников (reducers)** для случаев разных диаметров
4. **Если типы разные** (сварка → резьба):
   - Галочку не ставьте
   - Добавьте семейства **фитингов-переходников**
5. Нажмите **«Выбрать...»** — откроется окно выбора семейств из проекта
6. Укажите приоритет — фитинг с приоритетом 1 используется первым
7. Нажмите **«Сохранить»**

> **Важно:** Семейства фитингов должны быть **загружены в проект Revit**. SmartCon показывает только семейства с категорией «Трубы — фитинги» с ровно 2 коннекторами.

### Пример полной настройки для типичного проекта

**Типы коннекторов:**
| Код | Название |
|-----|----------|
| 1 | Сварка |
| 2 | Резьба |
| 3 | Раструб |

**Правила маппинга:**

| От | К | Фитинг? | Семейства |
|----|---|---------|-----------|
| Сварка (1) | Сварка (1) | Нет (Direct) | Reducer: «Муфта переходная», «Переход» |
| Резьба (2) | Резьба (2) | Нет (Direct) | Reducer: «Муфта переходная Р» |
| Раструб (3) | Раструб (3) | Нет (Direct) | Reducer: «Переход раструбный» |
| Сварка (1) | Резьба (2) | Да | «Переходник сварка-резьба», «Ниппель С-Р» |
| Сварка (1) | Раструб (3) | Да | «Переходник сварка-раструб» |
| Резьба (2) | Раструб (3) | Да | «Переходник резьба-раструб» |

> **Совет:** Если прямое правило отсутствует (например, «сварка → раструб»), SmartCon попытается найти **цепочку** фитингов через промежуточный тип (например, сварка → резьба → раструб), используя алгоритм поиска кратчайшего пути.

## PipeConnect — соединение двумя кликами

### Базовый сценарий соединения

**Что нужно:** Открытый 3D-вид в Revit, два элемента со свободными коннекторами (труба, кран, фитинг и т.д.).

**Пошагово:**

```
Шаг 1: Нажмите кнопку PipeConnect на ленте SmartCon
        └─ В строке состояния Revit появится:
           «PipeConnect: выберите ПЕРВЫЙ элемент (будет присоединён)»

Шаг 2: Кликните по ПЕРВОМУ элементу
        └─ Это элемент, который будет ПЕРЕМЕЩЁН к второму
        └─ Если у коннектора не задан тип — появится MiniTypeSelector
           для выбора типа соединения

Шаг 3: Кликните по ВТОРОМУ элементу
        └─ Это НЕПОДВИЖНЫЙ элемент, к которому присоединяют
        └─ Если у коннектора не задан тип — появится MiniTypeSelector

Шаг 4: SmartCon автоматически:
        ├─ Выравнивает первый элемент ко второму (позиция + поворот)
        ├─ Подбирает размер (если диаметры разные)
        ├─ Вставляет фитинг (если типы соединений разные)
        ├─ Вставляет переходник (если диаметры не совпали)
        └─ Открывает окно настройки

Шаг 5: В окне настройки вы можете:
        ├─ Повернуть элемент
        ├─ Сменить коннектор
        ├─ Выбрать другой фитинг
        ├─ Изменить размер
        └─ Включить перемещение цепочки

Шаг 6: Нажмите «Соединить» для подтверждения
        └─ или «Отмена» / ESC / закройте окно для отката
```

> **Любой шаг можно отменить.** Нажмите ESC в любой момент — все изменения откатятся.

### Окно настройки соединения (PipeConnectEditor)

После двух кликов автоматически открывается немодальное окно:

```
┌──────────────────────────────────────────────────┐
│ SmartCon - Настройка соединения            [X]   │
├──────────────────────────────────────────────────┤
│                                                  │
│ ── Коннектор ──────────────────────────────────  │
│ [▼ Connector 1 (Free) - Сварка]                 │
│ [Изменить]                                       │
│                                                  │
│ ── Поворот ────────────────────────────────────  │
│ [◀]  [ 0.00° ]  [▶]         Шаг: [15° ▼]       │
│                                                  │
│ ── Фитинги ────────────────────────────────────  │
│ ○ Без фитинга (прямое соединение)               │
│ ● СварнойШов DN50              [Примерить]      │
│ ○ Переходник_С-Р DN50          [Примерить]      │
│                                                  │
│ [ ] Переместить всю сеть                        │
│                                                  │
│ [ Отмена ]                  [ Соединить ]       │
└──────────────────────────────────────────────────┘
```

#### Секция «Коннектор»

| Элемент | Что делает |
|---------|-----------|
| Выпадающий список | Показывает все свободные коннекторы элемента. Выберите нужный — элемент переориентируется. |
| Кнопка «Изменить» | Открыть MiniTypeSelector для смены типа соединения. Если тип не задан — предложит выбрать. |

#### Секция «Поворот»

| Элемент | Что делает |
|---------|-----------|
| Кнопка ◀ | Повернуть элемент против часовой стрелки на шаг |
| Поле угла | Текущий угол поворота в градусах. Можно ввести точное значение и нажать Enter. |
| Кнопка ▶ | Повернуть элемент по часовой стрелке на шаг |
| Шаг поворота | Выберите из списка: 5°, 10°, 15°, 30°, 45°, 90° |

> Поворот выполняется вокруг оси коннектора. Вы видите результат в модели в реальном времени.

#### Секция «Фитинги»

| Элемент | Что делает |
|---------|-----------|
| «Без фитинга» | Прямое соединение коннекторов без промежуточного элемента |
| Список фитингов | Доступные фитинги из вашего маппинга. Рядом — DN (диаметр). |
| Кнопка «Примерить» | Вставляет фитинг в модель для предпросмотра. Старый фитинг (если был) удаляется. |

#### Нижняя панель

| Элемент | Что делает |
|---------|-----------|
| «Соединить» | Выполнить `ConnectTo` и зафиксировать все изменения. **Hotkey: Enter** |
| «Отмена» | Откатить ВСЕ изменения (позиция, размер, фитинги). **Hotkey: ESC** |
| «Переместить всю сеть» | Вместо одного элемента — переместить всю цепочку присоединённых элементов |
| Крестик [X] | То же, что «Отмена» — полный откат |

> **Одно действие = одна запись Undo в Revit.** Если нажали «Соединить», но передумали — Ctrl+Z откатит всё разом.

## Бизнес-кейсы — все сценарии

Ниже описаны все возможные сценарии использования PipeConnect с конкретными примерами.

### Кейс 1: Одинаковые диаметры и типы — прямое соединение

**Ситуация:** Труба DN50 (сварка) → Кран шаровой DN50 (сварка)

| Шаг | Что происходит |
|------|---------------|
| Клик 1 | Выбираете кран (он будет перемещён) |
| Клик 2 | Выбираете трубу (она неподвижна) |
| Автоматически | Диаметры совпадают (DN50 = DN50) |
| Автоматически | Типы совпадают (сварка = сварка) |
| Результат | Прямое соединение, **без фитинга** |
| Окно | В списке фитингов выбрано «Без фитинга» |
| Действие | Нажмите «Соединить» — готово |

**Что может пользователь:**
- Повернуть кран на нужный угол перед соединением
- Выбрать другой свободный коннектор крана
- Нажать «Отмена» — кран вернётся на место

---

### Кейс 2: Разные диаметры — автоподбор размера

**Ситуация:** Труба DN65 (сварка) → Кран шаровой DN50 (сварка)

**Вариант A: Точный типоразмер существует**

| Шаг | Что происходит |
|------|---------------|
| Клик 1 | Выбираете кран DN50 |
| Клик 2 | Выбираете трубу DN65 |
| Автоматически | SmartCon пытается переключить кран с DN50 на DN65 |
| Результат | Кран переключился на типоразмер DN65 — **автоподбор успешен** |
| Окно | Кран отображается DN65, статус: «АВТОПОДБОР (DN65)» |
| Действие | Нажмите «Соединить» — прямое соединение |

**Вариант B: Точного типоразмера нет**

| Шаг | Что происходит |
|------|---------------|
| Автоматически | Кран не удалось переключить на DN65 |
| Результат | Кран остаётся ближайшего размера (например, DN50) |
| Предупреждение | «Размер DN65 отсутствует. Выбран ближайший DN50, нужен переходник.» |
| Действие | См. Кейс 3 — будет вставлен переходник |

**Вариант C: Размер из LookupTable**

| Шаг | Что происходит |
|------|---------------|
| Автоматически | Семейство крана содержит таблицу размеров (LookupTable) |
| Результат | Формула решена, параметр записан, DN65 установлен точно |
| Действие | Прямое соединение без переходника |

---

### Кейс 3: Разные диаметры — автовставка переходника (Reducer)

**Ситуация:** Труба DN50 (сварка) → Кран DN40 (сварка), типоразмер DN50 не существует

| Шаг | Что происходит |
|------|---------------|
| Автоматически | Кран остаётся DN40 (автоподбор не удался) |
| Автоматически | DN50 ≠ DN40 → поиск переходника в маппинге |
| Результат | **Вставлен переходник DN50/DN40** между трубой и краном |
| Окно | Статус: «Вставлен переходник: [Имя] DN50→DN40» |
| Действие | Нажмите «Соединить» — соединение через переходник |

**Что нужно для работы:**
- В маппинге должно быть правило `Сварка → Сварка` с семействами-переходниками
- Семейство переходника должно быть загружено в проект
- У переходника должно быть 2 коннектора с разными DN

---

### Кейс 4: Разные типы соединений — обязателен фитинг

**Ситуация:** Труба DN50 (сварка, CTC=1) → Кран DN50 (резьба, CTC=2)

| Шаг | Что происходит |
|------|---------------|
| Клик 1 | Выбираете кран (резьба) |
| Клик 2 | Выбираете трубу (сварка) |
| Автоматически | Диаметры совпадают (DN50) |
| Автоматически | Типы НЕ совпадают (сварка ≠ резьба) |
| Автоматически | Поиск в маппинге: `Сварка → Резьба` |
| Результат | **Фитинг-переходник вставлен** (например, «Переходник сварка-резьба DN50») |
| Окно | В списке фитингов — варианты из маппинга |
| Действие | Выберите фитинг → «Примерить» → «Соединить» |

**Если фитинга для этой пары типов нет в маппинге:**
- SmartCon попытается найти **цепочку фитингов** через промежуточный тип
- Например: Сварка → Резьба (нет), но есть Сварка → Раструб → Резьба
- Будет вставлена цепочка из 2 фитингов

---

### Кейс 5: Разные диаметры + разные типы — фитинг + reducer

**Ситуация:** Труба DN65 (сварка) → Кран DN50 (резьба)

| Шаг | Что происходит |
|------|---------------|
| Автоматически | Попытка подогнать кран под DN65 (если возможно) |
| Автоматически | Вставка фитинга «сварка → резьба» |
| Автоматически | Фитинг подбирается: conn1 = DN65 (к трубе), conn2 = DN50 (к крану) |
| Вариант A | Точный типоразмер фитинга DN65/DN50 найден → готово |
| Вариант B | Фитинг DN65/DN40 (ближайший к DN50) → **+ reducer DN50/DN40** |
| Результат | `Труба ↔ Фитинг ↔ Reducer ↔ Кран` |
| Действие | Нажмите «Соединить» |

---

### Кейс 6: Цепочка элементов (Chain)

**Ситуация:** Нужно переместить и подключить не один элемент, а целую сеть:
Кран → Труба → Тройник → Отвод

| Шаг | Что происходит |
|------|---------------|
| Клик 1 | Выбираете кран (корневой элемент) |
| Клик 2 | Выбираете целевую трубу |
| В окне | Поставьте галочку **«Переместить всю сеть»** |
| Автоматически | SmartCon обходит все присоединённые элементы (BFS) |
| Автоматически | Все элементы перемещаются как **жёсткое тело** |
| Автоматически | На каждом уровне: выравнивание, подбор размера, ConnectTo |

**Что происходит внутри цепочки:**

```
Кран DN50 ──(соединяется с целевой трубой)
  └─ Труба DN50 ──(Disconnect → Align → AdjustSize → ConnectTo)
       └─ Тройник DN50/50/32 ──(Disconnect → Align → AdjustSize → ConnectTo)
            └─ Отвод DN32 ──(Disconnect → Align → AdjustSize → ConnectTo)
```

**Если размер элемента цепочки не совпадает с родителем:**
- SmartCon пытается автоподобрать размер
- Если не удалось — вставляет reducer

> **Совет:** Используйте Chain mode, когда нужно «подключить» целую ветку трубопровода к существующей сети одним действием.

---

### Кейс 7: Ручная смена размера в окне

**Ситуация:** Вы хотите изменить размер соединяемого элемента вручную.

| Действие | Результат |
|----------|-----------|
| Выберите другой DN из выпадающего списка | Элемент переключается на выбранный типоразмер |
| Новый DN ≠ DN целевой трубы | Автоматический поиск и вставка reducer |
| Reducer найден | Вставлен переходник, статус обновлён |
| Reducer не найден | Предупреждение, соединение без переходника |
| Вернуть «АВТОПОДБОР» | Размер возвращается к автоподобранному, reducer удаляется |

**Пример:** Труба DN50 ↔ Кран DN50 (автоподбор). Вы выбираете DN32:
1. Кран переключается на DN32
2. DN32 ≠ DN50 → поиск reducer
3. Вставлен «Муфта переходная DN50/DN32»
4. Нажмите «Соединить» — `Труба ↔ Reducer ↔ Кран`

---

### Кейс 8: Смена фитинга в окне

**Ситуация:** SmartCon предложил фитинг, но вы хотите другой.

| Действие | Результат |
|----------|-----------|
| Выберите другой фитинг из списка | Старый фитинг удаляется, новый вставляется и выравнивается |
| Нажмите «Примерить» | Фитинг реально вставляется в модель для предпросмотра |
| Выберите «Без фитинга» | Фитинг удаляется, прямое соединение, размер подгоняется |

---

### Кейс 9: Коннекторы без типа (пустой Description)

**Ситуация:** Вы впервые используете элемент, и у его коннектора ещё не задан тип соединения.

| Шаг | Что происходит |
|------|---------------|
| Клик по элементу | Обнаружен пустой Description |
| Автоматически | Рядом с курсором появляется **MiniTypeSelector** |
| MiniTypeSelector | Список типов из ваших настроек (Сварка, Резьба, ...) |
| Выбираете тип | Код записывается в Description семейства через EditFamily |
| Далее | Стандартный flow соединения |

> Тип соединения записывается **в семейство** (EditFamily), поэтому назначается один раз и сохраняется в RFA-файле. При следующем использовании этого семейства тип уже будет известен.

## Окно настроек — подробное описание

Окно настроек открывается кнопкой **Settings** на ленте. Оно состоит из двух вкладок.

### Вкладка «Типы коннекторов»

Управление типами соединений, которые используются в проекте.

| Действие | Как выполнить |
|----------|--------------|
| Добавить тип | Нажмите «+ Добавить», заполните код, название, описание |
| Удалить тип | Выделите строку, нажмите «Удалить» |
| Сохранить | Нажмите «Сохранить» — данные запишутся в **ExtensibleStorage** текущего `.rvt` (ADR-012). Для переноса в другой проект используйте **Экспорт** в JSON. |

> **Код** — это число, которое записывается в `Description` коннектора семейства. Рекомендуем: 1 = Сварка, 2 = Резьба, 3 = Раструб и т.д.

### Вкладка «Правила маппинга»

Определение, какие фитинги использовать для соединения разных типов.

**Создание правила:**

1. Выберите **«От типа»** и **«К типу»** из выпадающих списков
2. **Одинаковые типы** (например, Сварка → Сварка):
   - Поставьте «Совместимы без фитинга»
   - Добавьте семейства **переходников** для разных диаметров
3. **Разные типы** (например, Сварка → Резьба):
   - Не ставьте галочку
   - Добавьте семейства **фитингов-переходников**

**Добавление семейств:**

1. Нажмите **«Выбрать...»** — откроется окно выбора семейств
2. Слева — доступные семейства из проекта (фильтр:PipeFitting, 2 коннектора)
3. Переместите нужные направо кнопкой **▶▶ Добавить**
4. Упорядочьте приоритет кнопками **▲ Вверх** / **▼ Вниз** (верхний = приоритет 1)
5. Нажмите **«Подтвердить»**

> **Приоритет важен:** при наличии нескольких подходящих фитингов SmartCon выберет первый по приоритету.

**Хранение:** ExtensibleStorage текущего проекта Revit (`.rvt`) — ADR-012. Для резервной копии / переноса между проектами используйте **Экспорт / Импорт** (JSON).

## Частые вопросы (FAQ)

### Общие

**Q: Какие версии Revit поддерживаются?**
A: Revit 2019, 2020, 2021, 2022, 2023, 2024, 2025.

**Q: Нужны ли права администратора?**
A: Нет. Установщик и плагин работают в контексте пользователя (`%APPDATA%` — только для DLL плагина).

**Q: Как сменить язык интерфейса?**
A: Кнопка About → переключатель языка (RU/EN). Настройка сохраняется.

**Q: Где хранятся настройки?**
A: Типы коннекторов и правила маппинга хранятся в **ExtensibleStorage** текущего Revit-проекта (`.rvt`) — ADR-012. Это означает, что каждый проект имеет свой набор настроек, и они путешествуют вместе с файлом. Глобальные JSON-файлы (для резервной копии или переноса) создаются вручную через кнопки **Экспорт / Импорт**.

### PipeConnect

**Q: Можно ли отменить соединение?**
A: Да. Нажмите «Отмена» или ESC в окне настройки — все изменения откатятся. Если уже нажали «Соединить» — Ctrl+Z в Revit.

**Q: Что если у элемента несколько свободных коннекторов?**
A: SmartCon выбирает ближайший к точке клика. В окне настройки можно переключить на другой через выпадающий список.

**Q: Работает ли в планах/разрезах?**
A: PipeConnect предназначен для работы в **3D-видах**.

**Q: Что если фитинг не найден?**
A: SmartCon покажет предупреждение. Соединение будет выполнено без фитинга, но без вызова `ConnectTo`. Вы можете добавить нужное правило в Settings и повторить.

**Q: Как работает автоподбор размера?**
A: SmartCon проверяет: LookupTable → формулы параметров → смена типоразмера. Если ни один способ не сработал — используется ближайший размер + переходник.

### Типы коннекторов

**Q: Куда записывается тип соединения?**
A: В поле `Description` коннектора в семействе (через EditFamily). Назначается один раз, сохраняется в RFA.

**Q: Можно ли не использовать типы коннекторов?**
A: Можно, но тогда не будет работать автоподбор фитингов. Каждое соединение будет прямым.

**Q: Что если два проекта используют разные коды типов?**
A: Типы коннекторов (CTC) хранятся в самих семействах (в `Description` коннектора). Правила маппинга хранятся в ExtensibleStorage текущего Revit-проекта (ADR-012) — то есть каждый проект настраивается независимо. Для копирования настроек используйте **Экспорт → Импорт**.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Кнопки SmartCon не появились на ленте | Проверьте, что .addin-файл создан в `%APPDATA%\Autodesk\Revit\Addins\<версия>\`. Перезапустите Revit. |
| «Сначала настройте типы коннекторов в Настройках» | Откройте Settings → добавьте хотя бы один тип коннектора → Сохранить |
| «Нет свободных коннекторов» | У элемента все коннекторы заняты. Отсоедините нужный или выберите другой элемент. |
| Фитинг не вставляется | Проверьте: (1) семейство загружено в проект, (2) правило маппинга создано, (3) у семейства 2 коннектора |
| Переходник не вставляется | Проверьте: правило для одинаковых типов содержит семейства-переходники в настройках |
| Неверный тип соединения | Откройте Settings → измените код типа. Или используйте «Изменить» в окне настройки для пересмотра. |
| Плагин не загружается после обновления | Перезапустите Revit. Если не помогло — проверьте `%APPDATA%\SmartCon\<версия>\` на наличие DLL (папка установки плагина). |
