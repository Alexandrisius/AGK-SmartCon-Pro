# Business Case: Системные семейства в FamilyManager

## Статус
Черновик. Предварительный план после мозгового штурма.

---

## Проблема

FamilyManager (FM) на данный момент поддерживает только **загружаемые семейства** (.rfa), созданные в Редакторе семейств Revit. Однако в проектах активно используются **системные семейства** (стены, перекрытия, трубы, воздуховоды, лотки, короба, лестницы, крыши и др.), которые невозможно сохранить как .rfa и загрузить стандартным способом.

Инженерам-проектировщикам необходимо:
- Хранить библиотеку системных типов (типоразмеров) для быстрого переиспользования между проектами.
- Загружать системные типы в проект из каталога FM одной командой.
- Размещать элементы системных семейств (например, нарисовать трубу нужного типа) напрямую из FM.

---

## Решение

Расширить FamilyManager поддержкой системных типов Revit. Системные типы будут храниться в каталоге FM наравне с загружаемыми семействами, используя аналогичную модель: **проект .rvt = семейство, ElementType'ы внутри = типоразмеры**.

### Принцип работы

1. **Импорт**: пользователь открывает проект-источник (.rvt), выделяет несколько элементов в Revit — FM извлекает их системные типы и сохраняет как типоразмеры одного семейства.
2. **Хранение**: исходный проект (.rvt) сохраняется в managed storage. Имя семейства в FM = имя файла .rvt. В БД хранятся ссылки на типоразмеры (UniqueId) внутри этого проекта.
3. **Размещение**: при команде "Загрузить и разместить" FM копирует выбранный типоразмер из .rvt в активный проект и активирует стандартный инструмент Revit.

---

## Пользовательские сценарии

### UC-1: Импорт системного семейства в каталог

**Актор**: BIM-менеджер / инженер-проектировщик.

**Шаги**:
1. Пользователь открывает в Revit проект-источник с нужными системными типами (например, проект "Трубы.rvt" с 5 типами труб).
2. В FM нажимает стрелку рядом с кнопкой "Импорт" — выбирает "Импорт системного семейства".
3. FM переключается в режим пикера: пользователь выделяет несколько элементов в проекте.
4. FM извлекает уникальные типы выделенных элементов.
5. Создаётся семейство в FM с именем файла проекта ("Трубы").
6. Каждый уникальный тип становится типоразмером этого семейства.
7. Проект .rvt сохраняется в managed storage.
8. Семейство появляется в каталоге FM в категории "Без категории".

**Результат**: системное семейство с типоразмерами доступно в каталоге FM.

**Примечание**: Рекомендуется использовать "чистые" проекты (только типы, без модели) для минимизации размера файла.

---

### UC-2: Размещение типоразмера в проекте

**Актор**: Инженер-проектировщик.

**Шаги**:
1. Пользователь находит в каталоге FM системное семейство (например, "Трубы").
2. Раскрывает его и выбирает нужный типоразмер (например, "Медная труба").
3. Нажимает "Загрузить и разместить".
4. FM открывает .rvt из storage, находит тип по UniqueId.
5. Если тип с таким именем уже есть в активном проекте — FM активирует существующий (skip).
6. Если нет — FM копирует типоразмер через `ElementTransformUtils.CopyElements`.
7. Активируется стандартный инструмент Revit для размещения элементов данного типа.
8. Пользователь размещает элементы в проекте стандартным способом Revit.

**Результат**: элементы выбранного типоразмера размещены в проекте.

---

### UC-3: Извлечение атрибутов типоразмера

**Актор**: BIM-менеджер.

**Шаги**:
1. Пользователь выбирает типоразмер в каталоге FM.
2. Нажимает "Импорт данных" (существующая команда FM).
3. FM открывает .rvt из storage, находит тип по UniqueId.
4. Извлекает параметры типа в соответствии с настроенным пресетом категории.
5. Атрибуты отображаются в окне "Свойства" FM.

**Результат**: параметры типоразмера доступны для просмотра.

---

### UC-4: Дозагрузка типоразмеров (обновление семейства)

**Актор**: BIM-менеджер.

**Шаги**:
1. В проекте-источнике появились новые типы (или изменились существующие).
2. Пользователь повторно запускает "Импорт системного семейства".
3. Выделяет новые/изменённые элементы в проекте.
4. FM обнаруживает, что .rvt уже существует в storage (по SHA256 или пути).
5. Создаёт новую версию семейства: добавляет новые типоразмеры, обновляет существующие.
6. Старые типоразмеры, не выбранные при повторном импорте, сохраняются.

**Результат**: семейство в FM содержит актуальный набор типоразмеров.

---

## Ключевые решения

| Аспект | Решение |
|---|---|
| **Модель хранения** | Один .rvt файл = одно семейство в FM (аналог .rfa). |
| **Типоразмеры** | ElementType'ы внутри .rvt = типоразмеры семейства. |
| **Импорт** | Множественный выбор элементов в проекте. |
| **Имя семейства** | Из имени .rvt файла (без расширения). |
| **Идентификация типа** | `UniqueId` элемента (стабильный GUID). `ElementId` не используется. |
| **Хранение .rvt** | Файл сохраняется в managed storage в папке версии Revit (`R25/`, `R21/`). |
| **Чистота проекта** | Рекомендуется использовать "чистые" проекты (только типы, без модели). Регламент BIM-мастера. |
| **Копирование в проект** | `ElementTransformUtils.CopyElements` — только выбранный типоразмер. |
| **Конфликт имён** | Если тип с таким именем уже есть в проекте — пропускаем (skip), активируем существующий. |
| **Активация инструмента** | `UIDocument.PostRequestForElementTypePlacement(ElementType)`. |
| **Категории** | Один проект = одна категория (регламент). Функционального ограничения нет, валидатор предупреждает о смешанных категориях. |
| **Атрибуты** | Извлекаются отдельной командой "Импорт данных", не при импорте. |
| **Превью** | Не генерируется автоматически. Пользователь добавляет вручную, как у .rfa. |
| **Категории в FM** | Семейство загружается в "Без категории". Пользователь перемещает в кастомные категории. |

## Границы решения (Out of Scope)

- Автоматическое отслеживание изменений в шаблонных .rvt.
- Генерация превью/иконок для системных семейств.
- Отображение MEP-размеров (сегментов) как под-узлов в дереве FM.
- Предустановка диаметра MEP перед размещением (ограничение API Revit).
- Удаление дублирующихся материалов при копировании между проектами.
- Удаление отдельных типоразмеров из системного семейства (только полное обновление семейства).
- Функциональное ограничение "один проект = одна категория" (только регламент + валидатор).

---

## Критерии приёмки

1. Пользователь может импортировать несколько системных типов из открытого .rvt-проекта как типоразмеры одного семейства.
2. Имя семейства в FM соответствует имени .rvt файла.
3. Импортированное семейство отображается в каталоге FM с типоразмерами внутри.
4. Пользователь может разместить конкретный типоразмер в активном проекте.
5. Системный тип корректно копируется в проект вместе с зависимостями.
6. Если тип уже существует в проекте — FM активирует существующий, не создавая дубликатов.
7. Атрибуты типоразмера извлекаются командой "Импорт данных" и отображаются в окне свойств.
8. Повторный импорт того же .rvt создаёт новую версию семейства с дополнительными/обновлёнными типоразмерами.
9. Поддерживаются все основные системные семейства: стены, перекрытия, трубы, воздуховоды, лотки, короба, лестницы, крыши.
