## Шаблон проверки для каждого сценария

Для КАЖДОГО теста ниже — отметить результат по каждому критерию:

| Критерий | Описание | Pass / Fail / N/A |
|---|---|---|
| **C: Connect** | Соединение выполнено успешно (нет ошибки) | ☐ |
| **R: Radius** | Радиусы коннекторов совпадают (визуально и через `Lookup` параметров) | ☐ |
| **P: Position** | Позиция корректна — нет смещения Origin коннекторов (визуально, зазор ≈ 0) | ☐ |
| **O: Orientation** | Ориентация корректна — нет артефактов вращения (BasisZ антипараллельны) | ☐ |
| **U: Undo** | Ctrl+Z откатывает ВСЮ операцию (TransactionGroup.Assimilate → одна запись Undo) | ☐ |
| **E: Errors** | Нет неожиданных диалогов ошибок/предупреждений Revit | ☐ |

## 1. Базовые сценарии PipeConnect

### T-01: Прямое соединение (pipe-to-pipe, одинаковый диаметр)

**Предусловие:** Две трубы одного DN (напр. DN50), тип коннекторов совместим (IsDirectConnect).

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать первую трубу (dynamic) | Элемент подсвечен, приглашение выбрать второй |
| 2 | Выбрать вторую трубу (static) | Открывается PipeConnectEditor |
| 3 | Проверить: фитинг = «Прямое соединение» | Карточка «Прямое соединение» выбрана |
| 4 | Нажать «Соединить» | Элементы соединены, окно закрылось |

**Проверка:** C R P O U E

---

### T-02: Вставка фитинга (pipe-to-pipe, разные типы коннекторов → фитинг)

**Предусловие:** Две трубы одного DN, но разные типы коннекторов (напр. Сварка ↔ Фланец), в маппинге есть фитинг.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать первую трубу (dynamic) | — |
| 2 | Выбрать вторую трубу (static) | PipeConnectEditor открыт, карточка фитинга видна |
| 3 | Проверить: фитинг выбран автоматически | Карточка фитинга подсвечена |
| 4 | Нажать «Соединить» | Фитинг вставлен, соединение выполнено |

**Проверка:** C R P O U E

---

### T-03: Вставка переходника (pipe-to-pipe, разный диаметр → reducer)

**Предусловие:** Труба DN50 + труба DN32, одинаковый тип коннектора.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать DN32 (dynamic) | — |
| 2 | Выбрать DN50 (static) | PipeConnectEditor открыт, виден переходник |
| 3 | Проверить: Reducer ComboBox видим | IsReducerVisible = true |
| 4 | Нажать «Соединить» | Переходник вставлен, оба конца соединены |

**Проверка:** C R P O U E

---

### T-04: Цепочка (3+ элемента последовательно)

**Предусловие:** 3+ последовательно соединённых трубы + отдельный static-элемент. При выборе dynamic строится ConnectionGraph с уровнями.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать первый элемент цепочки (dynamic) | — |
| 2 | Выбрать static | PipeConnectEditor открыт, кнопки Chain+/Chain− видимы |
| 3 | Нажать Chain+ (Уровень 1) | Первый элемент цепочки присоединён |
| 4 | Нажать Chain+ (Уровень 2) | Второй элемент цепочки присоединён |
| 5 | Нажать «Соединить» | Вся цепочка зафиксирована, одна запись Undo |

**Проверка:** C R P O U E

---

### T-05: Циклическое переключение коннектора

**Предусловие:** Dynamic-элемент с 3+ свободными коннектороми (тройник, крест).

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать dynamic (многопортовый) | — |
| 2 | Выбрать static | PipeConnectEditor открыт |
| 3 | Нажать «Сменить коннектор» | Активный коннектор переключился на следующий |
| 4 | Нажать ещё раз | Следующий непосещённый коннектор |
| 5 | Все посещены → нажать ещё раз | Цикл начинается сначала |
| 6 | Нажать «Соединить» | Соединение с текущим коннектором |

**Проверка:** C R P O U E

---

### T-06: Смена динамического размера (через ComboBox)

**Предусловие:** Dynamic-элемент — FamilyInstance с LookupTable и несколькими типоразмерами.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать dynamic | — |
| 2 | Выбрать static | PipeConnectEditor открыт |
| 3 | Открыть ComboBox размеров | Виден «АВТОПОДБОР» + список типоразмеров |
| 4 | Выбрать другой DN | Размер изменён, позиция скорректирована |
| 5 | Нажать «Соединить» | Соединение с новым размером |

**Проверка:** C R P O U E

---

### T-07: Переназначение CTC (кнопка CTC)

**Предусловие:** Вставлен фитинг или переходник.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выбрать dynamic | — |
| 2 | Выбрать static | PipeConnectEditor открыт |
| 3 | Дождаться вставки фитинга | Фитинг виден |
| 4 | Нажать кнопку «CTC» фитинга | Открывается окно назначения типов коннекторов |
| 5 | Изменить тип для одного коннектора | — |
| 6 | Подтвердить | Фитинг переориентирован, окно закрылось |
| 7 | Нажать «Соединить» | Соединение с новым CTC |

**Проверка:** C R P O U E

---

### T-08: Отмена операции (Ctrl+Z → полный откат)

**Предусловие:** Любой активный сценарий с изменениями.

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выполнить шаги T-01..T-03 до «Соединить» | Модель изменена (фитинг вставлен, элементы сдвинуты) |
| 2 | Нажать «Отмена» в PipeConnectEditor | Окно закрылось, группа откачена |
| 3 | Проверить модель | Все изменения откатаны: элементы на месте, фитингов нет |
| 4 | Нажать Ctrl+Z в Revit | Ничего не происходит (нет записей Undo) |

**Вариант 2 (Undo после Connect):**

| Шаг | Действие | Ожидание |
|---|---|---|
| 1 | Выполнить Connect до конца | Успешное соединение |
| 2 | Нажать Ctrl+Z | Вся операция откатана (одна запись Undo) |
| 3 | Проверить модель | Элементы возвращены к исходному состоянию |

**Проверка:** C U E

## 2. Краевые случаи

### T-10: Разные материалы труб

**Предусловие:** Труба Сталь DN50 + труба Медь DN50 (разные Material).

**Проверка:** C R P O U E. Убедиться, что радиусы совпадают (разные материалы → могут быть разные толщины стенок, но R — внутренний).

---

### T-11: FlexPipe (гибкая труба)

**Предусловие:** FlexPipe DN50 + обычная труба DN50.

**Проверка:** C R P O U E. FlexPipe корректно перемещается (MoveElement, не LocationCurve).

---

### T-12: Элемент под углом (не коаксиальные)

**Предусловие:** Труба по оси X + труба по оси Y (угол 90°). Коннекторы направлены навстречу.

**Проверка:** C R P O U E. Поворот и перемещение корректны, нет «загиба» трубы.

---

### T-13: Многопортовый фитинг (тройник / крест)

**Предусловие:** Тройник с 3 коннекторами, dynamic = тройник.

**Проверка:**
- Cycle Connector переключает между 3 коннекторами (T-05)
- При Connect → только выбранный коннектор соединён
- Остальные коннекторы не нарушены
- C R P O U E

---

### T-14: Dynamic family с LookupTable

**Предусловие:** Dynamic = FamilyInstance с size_lookup в формуле параметра.

**Проверка:**
- ComboBox показывает типоразмеры из таблицы
- Выбор размера → TrySetConnectorRadius через query params
- Если точного DN нет → вставляется reducer
- C R P O U E

---

### T-15: Dynamic family без LookupTable

**Предусловие:** Dynamic = FamilyInstance с Instance-параметром Radius (без таблицы).

**Проверка:**
- Прямая запись TrySetConnectorRadius
- Если TrySet → false → reducer needed
- C R P O U E

## 3. Регрессионные проверки инвариантов

### I-03: Транзакции через сервис

**Ручная проверка:** Поиск по логам строки `"new Transaction"` — не должно быть за пределами `RevitTransactionService`.

### I-04: TransactionGroup Assimilate

**Ручная проверка:** После Connect → Ctrl+Z → ровно ОДИН шаг Undo (не несколько).

### I-05: Хранение ссылок

**Ручная проверка:** Между транзакциями — только `ElementId`. Открыть лог, убедиться что `RefreshConnector` вызывается в начале каждой транзакции.

### I-09: Чистота Core

**Автоматизировано:** `scripts/validate-refactoring.ps1` — Check #3.
