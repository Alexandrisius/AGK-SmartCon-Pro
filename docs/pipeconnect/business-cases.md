# PipeConnect: Бизнес-кейсы и ожидаемая логика

> Дата: 2026-04-08
> Назначение: Единый справочник «ситуация → ожидаемое поведение» для модуля PipeConnect.
> Загружать: при реализации логики соединения, подбора фитингов, reducer, UI-взаимодействия.

---

## Общие определения

| Термин | Значение |
|---|---|
| **Static** | Неподвижный элемент (второй клик). К нему присоединяют. |
| **Dynamic** | Движущийся элемент (первый клик). Его перемещают и подгоняют. |
| **CTC** | ConnectionTypeCode — тип соединения (1=Сварка, 2=Резьба и т.д.) |
| **Reducer** | Переходник сечения — фитинг с двумя коннекторами разных DN одного CTC |
| **Фитинг** | Переходник по типу соединения (сварка→резьба) или просто соединительная деталь |
| **DirectConnect** | Прямое соединение без промежуточного фитинга |
| **S4** | Шаг подбора параметров (радиусов) — см. `state-machine.md` |
| **S5** | Шаг подбора фитингов — см. `state-machine.md` |
| **Chain** | Цепочка элементов, присоединённых к dynamic (кнопки +/− в UI) |

## Кейс 1: Радиусы совпадают, CTC совпадают — прямое соединение

**Пример:** Труба DN50 (сварка) → Кран шаровой DN50 (сварка)

| Шаг | Что происходит |
|---|---|
| S4 | Радиусы совпадают → `Skip=true` |
| S5 | CTC совпадают → `IsDirectConnect=true`, фитинг не нужен |
| UI | Выпадающий список: «Без фитинга (прямое соединение)» |
| Init | Выравнивание dynamic к static. Размер не меняется. |
| Соединить | `ConnectTo(static, dynamic)` → Assimilate |

**Вариант пользователя — смена размера:**
Пользователь выбирает DN40 в выпадающем списке.

| Шаг | Что должно произойти |
|---|---|
| ChangeDynamicSize | `TrySetConnectorRadius(DN40)` → кран меняется на DN40 |
| Система | DN40 ≠ DN50 → **вставить reducer DN50/DN40** (если настроен в маппинге) |
| UI | StatusMessage: «Вставлен переходник: [Имя] DN50→DN40» |
| Соединить | `ConnectTo(static, reducer.conn1)` + `ConnectTo(reducer.conn2, dynamic)` |

> **Важно:** При нажатии «Соединить» размер dynamic **НЕ** подгоняется обратно под static.

---

## Кейс 2: Радиусы НЕ совпадают, CTC совпадают — автоподбор + возможный reducer

**Пример:** Труба DN65 (сварка) → Кран шаровой DN50 (сварка)

| Шаг | Что происходит |
|---|---|
| S4 | DN50 ≠ DN65 → `TrySetConnectorRadius(DN65)` — попытка подогнать кран |
| S4.result | **Успех** — кран переключился на типоразмер DN65 |
| S5 | CTC совпадают → DirectConnect |
| UI | Кран отображается DN65, пользователь видит «АВТОПОДБОР (DN65)» |
| Соединить | Прямое соединение |

**Вариант S4: точного типоразмера нет:**

| Шаг | Что происходит |
|---|---|
| S4 | DN50 ≠ DN65 → `TrySetConnectorRadius(DN65)` → **не удалось** |
| S4 | Применяется ближайший типоразмер → `ExpectNeedsAdapter=true` |
| S5 | CTC совпадают → DirectConnect |
| UI | Предупреждение: «Размер DN65 отсутствует. Выбран ближайший DN50, нужен переходник.» |
| Init | Кран остаётся ближайшего размера |
| Соединить | **Вставить reducer** между static и dynamic (если настроен) |

**Вариант S4: LookupTable с точным совпадением:**

| Шаг | Что происходит |
|---|---|
| S4 | LookupTable содержит DN65 → формула решена, параметр записан |
| S4 | `ExpectNeedsAdapter=false` |
| Init | Кран DN65, точное совпадение |
| Соединить | Прямое соединение |

## Кейс 3: CTC НЕ совпадают — обязателен фитинг-переходник

**Пример:** Труба DN50 (сварка, CTC=1) → Кран шаровой DN50 (резьба, CTC=2)

| Шаг | Что происходит |
|---|---|
| S4 | Радиусы совпадают → `Skip=true` |
| S5 | CTC 1≠2 → поиск в маппинге `GetMappings(1, 2)` |
| S5 | Найдено правило: `{FromType=1, ToType=2, FittingFamilies=[...]}` |
| UI | Выпадающий список: фитинги из правила (напр. «Переходник сварка-резьба») |
| Init | Фитинг вставлен, выровнен, размер подобран |
| Соединить | `ConnectTo(static, fitting.conn1)` + `ConnectTo(fitting.conn2, dynamic)` |

**Вариант: радиусы НЕ совпадают + CTC НЕ совпадают:**

**Пример:** Труба DN65 (сварка) → Кран шаровой DN50 (резьба)

| Шаг | Что происходит |
|---|---|
| S4 | DN50 ≠ DN65 → подгонка крана под DN65 (если возможно) |
| S5 | CTC 1≠2 → фитинг-переходник вставляется |
| Фитинг | `TrySetFittingTypeForPair` подбирает типоразмер: conn1=DN65 (static), conn2=DN50 (dynamic) |
| Init | Фитинг DN65/DN50 вставлен между элементами |
| Соединить | Через фитинг |

**Вариант: точного типоразмера фитинга нет:**

| Шаг | Что происходит |
|---|---|
| S5 | Фитинг вставлен, `TrySetFittingTypeForPair` → ближайший к dynamic |
| Результат | conn1=DN65 (exact static), conn2=DN40 (ближайший к DN50) |
| Система | DN40 ≠ DN50 (dynamic) → **вставить reducer DN50/DN40 между фитингом и краном** |
| Соединить | static ↔ fitting ↔ reducer ↔ dynamic |

## Кейс 4: Пользователь меняет размер в UI (выпадающий список)

### 4A: Прямое соединение (без фитинга)

**Начальное состояние:** Труба DN50 (static) ↔ Кран DN50 (dynamic, подобран автоподбором)

**Действие:** Пользователь выбирает DN32 в выпадающем списке.

| Шаг | Что должно произойти |
|---|---|
| 1 | `ChangeDynamicSize()` → `TrySetConnectorRadius(DN32)` → кран → DN32 |
| 2 | Коррекция позиции (геометрия могла сместиться) |
| 3 | `_userChangedSize = true` |
| 4 | Проверка: DN32 ≠ DN50 → поиск reducer в маппинге |
| 5 | **Reducer найден** → вставка DN50/DN32, выравнивание, обновление UI |
| 6 | **Reducer НЕ найден** → предупреждение в StatusMessage, без ошибки |

**Далее «Соединить»:**
- Если reducer вставлен: `static ↔ reducer ↔ dynamic`
- Если reducer не найден: прямое соединение с несовпадающими радиусами (предупреждение)

### 4B: Соединение через фитинг

**Начальное состояние:** Труба DN50 (сварка) ↔ Фитинг-переходник ↔ Кран DN50 (резьба)

**Действие:** Пользователь выбирает DN32 для крана.

| Шаг | Что должно произойти |
|---|---|
| 1 | `ChangeDynamicSize()` → кран → DN32 |
| 2 | `InsertFittingSilentNoDynamicAdjust()` → фитинг переподбирается |
| 3 | `SizeFittingConnectors`: `TrySetFittingTypeForPair(conn1=DN50, conn2=DN32)` |
| 4a | **Точный типоразмер найден** (conn1=DN50, conn2=DN32) → фитинг обновлён |
| 4b | **Точного нет** → ближайший к DN32 (напр. conn2=DN25) + **reducer DN32/DN25** между фитингом и краном |
| 5 | Выравнивание, обновление UI |

### 4C: Возврат к АВТОПОДБОР

**Действие:** Пользователь выбирает «АВТОПОДБОР» в выпадающем списке.

| Шаг | Что должно произойти |
|---|---|
| 1 | `_userChangedSize = false` |
| 2 | Размер dynamic возвращается к статическому радиусу (как при Init) |
| 3 | Если был вставлен reducer — удалить |
| 4 | UI обновляется |

## Кейс 5: Цепочка элементов (Chain+)

**Пример:** Dynamic = кран DN50, цепочка = труба DN50 → тройник DN50/50/32 → отвод DN32

**Chain+ уровень 1 (труба DN50):**

| Шаг | Что происходит |
|---|---|
| a | Disconnect трубы от всех соседей |
| b | Найти ребро к родителю (кран) |
| c | AdjustSize: R(труба) == R(кран) = DN50 → совпадает, подгонка не нужна |
| d | Align: труба выравнивается к коннектору крана |
| e | ConnectTo: кран ↔ труба |

**Chain+ уровень 2 (тройник DN50/50/32):**

| Шаг | Что происходит |
|---|---|
| a | Disconnect тройника |
| b | Найти ребро: parent = труба (DN50) |
| c | AdjustSize: тройник parent-facing = DN50 → совпадает → ОК |
| d | Align + ConnectTo |

**Chain+ уровень 3 (отвод DN32, но parent = тройник conn3 = DN32):**

| Шаг | Что происходит |
|---|---|
| c | AdjustSize: отвод DN32 == тройник conn3 DN32 → совпадает → ОК |

**Вариант: размер не совпадает (отвод DN25 вместо DN32):**

| Шаг | Что происходит |
|---|---|
| c.1 | `TrySetConnectorRadius(отвод, connIdx, DN32)` → пытаемся подогнать |
| c.3 | Верификация: DN25 ≠ DN32 → подгонка не удалась |
| c.3 | **InsertReducer(parent=DN32, child=DN25)** → reducer DN32/DN25 |
| d | Align: отвод выравнивается к **conn2 reducer'а** (DN25) |
| e | ConnectTo: тройник ↔ reducer ↔ отвод |

## Кейс 6: Пользователь переключает фитинг в UI

**Начальное состояние:** Труба DN50 (сварка) ↔ Кран DN50 (резьба) — выбран фитинг «Ниппель сварка-резьба»

**Действие:** Пользователь выбирает другой фитинг из списка (напр. «Муфта переходная»).

| Шаг | Что должно произойти |
|---|---|
| 1 | Текущий фитинг удаляется из модели |
| 2 | Новый фитинг вставляется, выравнивается к static |
| 3 | `SizeFittingConnectors` подбирает типоразмер |
| 4 | Dynamic выравнивается к conn2 нового фитинга |
| 5 | UI обновляется |

**Действие:** Пользователь выбирает «Без фитинга (прямое соединение)».

| Шаг | Что должно произойти |
|---|---|
| 1 | Текущий фитинг удаляется |
| 2 | Dynamic выравнивается непосредственно к static |
| 3 | Размер dynamic подгоняется под static (автоподбор) |
| 4 | `_userChangedSize = false` |
| 5 | Если был reducer — удалить |

## Кейс 7: Reducer в маппинге — настройка и поиск

### 7.1 Где настраиваются reducer-ы

Reducer-ы хранятся в `FittingMappingRule.ReducerFamilies` — JSON в AppData.

**Пример JSON:**
```json
{
  "FromType": { "Value": 1 },
  "ToType": { "Value": 1 },
  "IsDirectConnect": true,
  "FittingFamilies": [],
  "ReducerFamilies": [
    { "FamilyName": "Муфта переходная", "SymbolName": "*", "Priority": 1 },
    { "FamilyName": "Переход", "SymbolName": "*", "Priority": 2 }
  ]
}
```

**Ключевое условие:** `FromType == ToType` — reducer соединяет **один и тот же тип** коннектора (напр. сварка↔сварка), но **разные DN**.

### 7.2 Как система ищет reducer

1. `FittingMapper.GetMappings(parentCTC, parentCTC)` — правила где FromType == ToType == parentCTC
2. Из первого правила берём `ReducerFamilies[0]` (по Priority)
3. `NetworkMover.InsertReducer()` — вставляет, выравнивает, подбирает типоразмер

### 7.3 Когда reducer НЕ будет найден

- В маппинге нет правила с `FromType == ToType` и `ReducerFamilies.Count > 0`
- Семейство reducer'а не загружено в проект Revit
- Ни один типоразмер reducer'а не подходит по размерам

**Поведение при отсутствии:** Предупреждение пользователю, соединение без reducer.

## Кейс 8: Все элементы с Description без CTC

**Ситуация:** Коннекторы не имеют ConnectionTypeCode (Description пустой).

| Шаг | Что происходит |
|---|---|
| S1.1 | MiniTypeSelector — пользователь выбирает тип для dynamic |
| S2.1 | MiniTypeSelector — пользователь выбирает тип для static |
| Далее | Стандартный flow по кейсам 1-7 |

**Важно:** Если CTC не задан для фитинга — система показывает диалог `FittingCtcSetupView` для назначения типов коннекторов перед вставкой.

## Кейс 9: Отмена (RollBack)

**Любое действие в PipeConnectEditor** может быть отменено:

| Кнопка | Что происходит |
|---|---|
| Отмена / ESC / Закрыть окно | `TransactionGroup.RollBack()` → все изменения откатываются |
| | Удалённые фитинги восстанавливаются |
| | Изменённые размеры возвращаются |
| | Вставленные reducer-ы удаляются |
| | Элементы возвращаются на исходные позиции |

**Chain− (откат уровня цепочки):**

| Шаг | Что происходит |
|---|---|
| a | Disconnect элементов уровня |
| b | Удалить reducer-ы, вставленные при Chain+ |
| c | Восстановить snapshot: размер, позиция, ориентация, FamilySymbol |
| d | Восстановить исходные соединения с соседями |

## Матрица решений: что происходит при несовпадении

| Ситуация | Autopodbor | Reducer | Фитинг | Соединить |
|---|---|---|---|---|
| DN совпадают, CTC совпадают | — | — | — | Прямое |
| DN не совпадают, CTC совпадают, автоподбор OK | dynamic → static DN | — | — | Прямое |
| DN не совпадают, CTC совпадают, автоподбор FAIL | ближайший DN | **вставить** | — | Через reducer |
| DN совпадают, CTC не совпадают | — | — | **вставить** | Через фитинг |
| DN не совпадают, CTC не совпадают, фитинг точный | dynamic → фитинг.conn2 | — | **вставить** | Через фитинг |
| DN не совпадают, CTC не совпадают, фитинг ближайший | — | **вставить** | **вставить** | фитинг + reducer |
| Пользователь сменил размер (direct) | — | **вставить** | — | Через reducer |
| Пользователь сменил размер (фитинг) | фитинг переподбирается | если не точный — **вставить** | обновить | фитинг ± reducer |

## Требования к маппингу для работы reducer

Для корректной работы автоматической вставки reducer'ов пользователь **должен** настроить:

1. **ConnectorTypes** — минимум один тип (напр. 1=Сварка, 2=Резьба)
2. **Правило маппинга** с `FromType == ToType` (один CTC) и `IsDirectConnect=true`
3. **ReducerFamilies** в этом правиле — список семейств-переходников
4. Семейства reducer'ов должны быть **загружены в проект Revit**
5. Семейство reducer'а должно иметь **2 коннектора** и несколько типоразмеров с разными DN

**Если редьюсер не настроен:** Система подгоняет dynamic под static или выбирает ближайший размер. Предупреждение пользователю.
