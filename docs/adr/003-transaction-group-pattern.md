# ADR-003: TransactionGroup + Assimilate для PipeConnect

**Статус:** accepted
**Дата:** 2026-03-25

## Контекст

Операция PipeConnect включает множество шагов: выравнивание, подбор параметров, вставка фитинга, поворот, ConnectTo. Каждый шаг — отдельная Transaction. Пользователь должен иметь возможность отменить всю операцию одним Ctrl+Z или откатить все изменения кнопкой «Отмена».

## Решение

Обернуть всю операцию PipeConnect в `TransactionGroup`:

1. TransactionGroup открывается в начале S3 (AligningConnectors)
2. Каждое действие — отдельная Transaction внутри группы
3. «Соединить» -> `TransactionGroup.Assimilate()` — все Transaction сливаются в одну Undo-запись
4. «Отмена» -> `TransactionGroup.RollBack()` — полный откат всех изменений

### Преимущества паттерна

- Пользователь видит реальные изменения в модели (фитинги, повороты) как превью
- При отмене — чистый откат, никакого мусора
- Одна запись в Undo-стеке при подтверждении

## Последствия

**Плюсы:**
- Чистая отмена всей операции
- Реальное превью (не overlay)
- Одна запись Undo

**Минусы:**
- TransactionGroup держит модель «заблокированной» пока открыто окно
- Нужно аккуратно обрабатывать крайние случаи (закрытие окна, crash)

## Альтернативы

1. **Одна большая Transaction:** нет возможности интерактивного превью.
2. **Отдельные Transaction + ручной Undo:** сложно и ненадёжно.
3. **Ghost/Overlay preview:** не требует транзакций, но сложная реализация визуализации без DirectContext3D.
