# ADR-002: Хранение типа соединения в Connector.Description

**Статус:** accepted
**Дата:** 2026-03-25

## Контекст

Revit не имеет встроенной системы типов соединений для коннекторов (сварка, резьба, раструб и т.д.). Нужно где-то хранить код типа, привязанный к конкретному коннектору семейства.

## Решение

Использовать поле `Connector.Description` для хранения числового кода типа соединения (`ConnectionTypeCode`).

- Код — целое число (1 = сварка, 2 = резьба, 3 = раструб, ...)
- Пустой/невалидный Description = тип не задан (`ConnectionTypeCode.Undefined`)
- Справочник типов хранится в JSON (AppData) и настраивается через UI

### Запись Description

Description — атрибут **семейства** (не экземпляра). Для записи:
1. `doc.EditFamily(family)` -> FamilyDocument
2. Найти ConnectorElement по индексу
3. Установить Description
4. `familyDoc.LoadFamily(doc, FamilyLoadOptions)` -> перезагрузить в проект

### MiniTypeSelector

При клике на коннектор с пустым Description — всплывает компактное окно рядом с курсором для выбора типа из справочника.

## Последствия

**Плюсы:**
- Тип привязан к коннектору семейства, а не к экземпляру
- Сохраняется при копировании элементов
- Видно в Revit Lookup для отладки

**Минусы:**
- EditFamily — медленная операция (открытие/закрытие семейства)
- Меняет семейство для всех экземпляров (это и плюс, и минус)

## Альтернативы

1. **Shared Parameter на экземпляре:** быстрее запись, но нужно заполнять для каждого экземпляра и не привязан к конкретному коннектору.
2. **Extensible Storage:** привязан к элементу, но сложнее и не переносится между проектами автоматически.
3. **Внешняя БД (SQLite):** полный контроль, но дополнительная зависимость и синхронизация.
