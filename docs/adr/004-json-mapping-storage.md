# ADR-004: JSON в AppData для хранения маппинга фитингов

**Статус:** superseded by [ADR-012](012-per-project-extensible-storage.md)
**Дата:** 2026-03-25
**Superseded:** 2026-04-19

> **Актуальное решение — ADR-012.** Маппинг теперь хранится в
> `ExtensibleStorage.DataStorage` каждого проекта. Глобальный JSON
> `%APPDATA%\AGK\SmartCon\connector-mapping.json` больше не читается
> автоматически; он используется только как источник для ручного
> Импорта через окно Settings.

## Контекст

Маппинг типов коннекторов на семейства фитингов — конфигурация уровня компании. Нужно хранилище, доступное из любого проекта Revit, простое в редактировании и переносе.

## Решение

Два JSON-файла в `%APPDATA%/SmartCon/`:

- `connector-types.json` — справочник типов коннекторов
- `fitting-mapping.json` — правила маппинга фитингов

### Формат connector-types.json

```json
[
  { "code": 1, "name": "Сварка", "description": "Сварное соединение" },
  { "code": 2, "name": "Резьба", "description": "Резьбовое соединение" },
  { "code": 3, "name": "Раструб", "description": "Раструбное соединение" }
]
```

### Формат fitting-mapping.json

```json
[
  {
    "fromTypeCode": 1,
    "toTypeCode": 1,
    "isDirectConnect": true,
    "fittingFamilies": [
      { "familyName": "СварнойШов", "symbolName": "*", "priority": 1 }
    ]
  },
  {
    "fromTypeCode": 1,
    "toTypeCode": 2,
    "isDirectConnect": false,
    "fittingFamilies": [
      { "familyName": "Переходник_С-Р", "symbolName": "*", "priority": 1 }
    ]
  }
]
```

### Сериализация

`System.Text.Json` (встроен в .NET 8). Без внешних зависимостей.

## Последствия

**Плюсы:**
- Глобальный на уровне компании (не привязан к .rvt)
- Легко редактировать вручную (JSON — человекочитаемый)
- Легко переносить между машинами (скопировать папку)
- Нет зависимости от БД

**Минусы:**
- Нет конкурентного доступа (один пользователь — ОК, сетевое хранилище — нет)
- Нет версионирования (решается через Git или ручной бэкап)

## Альтернативы

1. **SQLite:** Полноценная БД, но избыточна для конфига из 10-50 правил.
2. **Extensible Storage в проекте:** Привязано к .rvt, не глобальное.
3. **Реестр Windows:** Неудобно для структурированных данных.
